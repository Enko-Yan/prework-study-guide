HTML

  • The head element contains information about the webpage.
  • Information about the webpage is called Metadata
  • Charset specifies the character encoding for the HTML document.
  • The http-equiv element sets an HTTP header for content.
  • The viewport is the user's visible area of a web page. Varies with device. Smaller on phone than on computer.
  • The width=device-width part sets the width of the page to follow the screen-width of the device (which will vary depending on the device). The initial-scale=1.0 part sets the initial zoom level when the page is first loaded by the browser.
  • UTF-8 is a character encoding standard used for electronic communication. Defined by the Unicode Standard, the name is derived from Unicode Transformation Format - 8-bit.Almost every webpage is stored in UTF-8.
  • The h1, h2, h3, h4, h5, h6 elements represent the level of heading.
  • The p element represents a paragraph or block of text.
  • The ul, ol, li elements represent unordered lists, ordered lists, and list items.
  • The img element contains information about images that are displayed on the webpage.
  • The img element can contain various attributes such as src is the most important attribute because it defines the location of an image.
  • The br element creates a line of empty space, or a line break between two blocks of content.
  • The a element (anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The body element represents the visible content shown to the user.
  • The script element embeds client side scripts in the HTML document.
  • In semantic HTML elements the name describes it's content and role, such as body element. Use these when possible as they make HTML easier to read.
  • Examples of non semantic elements include div which creates a division in the page or span to create an inline division.
  • The id attribute is used by HTML to identify elements and select them.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is used to style an element directly in the HTML file. Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. Useful for quick permanent changes.
  • An example of Inline CSS would be using the style attribute to change the color of a single h1 heading.
  • You can embed an entire stylesheet directly into an HTML file using a style element in the head element. This internal stlye sheet contains CSS rules for the webpage in the head section of the HTML file. This rule would onlt apply to that page.
  • External CSS style sheets are the most common way to style a webpage. It allows developers to keep all their CSS rules in a separate file making design changes easier.
  • When using a external style sheet we link the file using a link element placed in the HTML's head.
  • Separation of concerns is a term referring to seperating code into different files based on use. Based on the idea that each section of code should have it's own responsibility.
  • Declarations contain two components the CSS property we want to apply and the value of the property. color: blue; the property is color and the value is blue.
  • The margin property indicates how much space we want around the outside of an element. The margin is not visible on the page, it only adds a buffer.
  • The padding property adds space around the content inside an element. When you increase the padding you make the entire element bigger. The padding is transparent but is located inside the element.
  • The * selector applies a rule to all elements visible on the page.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creats a new branch and switches to it.
  • git checkout main switches to the main branch.
  • git add -A The add command adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes.
  • git commit -m "descriptive message" commit your staged content as a new commit snapshot.
  • git pull fetch and merge any commits from the tracking remote branch. Such as the command git pull origin main would receive a branch's modifications into the local envirnment. The origin keyword indicates the source of the pull such as the GitHub repo.
  • git push would send the commits to the GitHub repo.

JavaScript

  • Control flow is the order in which a computer executes script. JavaScript runs from the first statement at the top to the last at the bottom.
  • Conditional statements and loops can be used to control in which code is executed. Or executed at all.
  • Falsy if it is one of the following values 0, -0, On, "", null, undefined, NaN or a Boolean data type with the value of false.
  • All other values, including a Boolean with the value if true, are considered truthy.
  • Conditional statements "if", "else if"
  • The "===" checks to see if two values are equal.
  • Arrays store multiple data types in one variable.
  • An Index is a unique number assigned to different data in an array and is used to call a specific item within the variable. The first item is 0 then proceeds 1,2,3 etc.
  • const assigns a variable that cannot be changed.
  • The let keyword followed by a unique varible name and the assignment operator = to assign a value to a varible. let allows us to reassign a varible over time.
  • x++ would be a short way of writing x+1
  • for loop repeats code until prompted to stop.
